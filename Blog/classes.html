Classes and a bit about Object Orientated Programming

11 January, 2015 by Sandra Muller in Blog Posts


OOP or Object Orientated Programming

OOP is a programming style based on the concept of "objects". Objects are data structures that contain data often know as attributes and code in the form of procedures also know as methods. So imagine you had many containers and you wanted to access elements in those containers. You could open them up and do what you need to with the contents and then move on. That's a very simple explanation and outline, but it gives you a basic understanding. Programming moved towards this method of "putting things into containers" so that the code was easily accessable and grouped nicely - ultimately organized better. Before OOP, code was extremely long and you would have to go through lines of it. Now if you know what object you are looking for you can find all the code belonging to that object easily. The most popular method for creating objects is by using classes, meaning that objects are instances of classes.

Classes

Based on what was said above, a class is a group of related methods and variables. Methods are functions that take an input and return a result. Variables are properties of the class.
