My cheat sheet for SQL

27 January, 2015 by Sandra Muller in Blog Posts


I typed up this cheat sheet while working through some exercises on SQLZoo, so most of the examples are specific to the tables in the exercises.


SELECT, FROM, WHERE & LIKE


General SELECT Query:

SELECT ... FROM ... WHERE ... = ' '

e.g. SELECT population FROM world WHERE name = 'Germany'

  • population, and name are columns, Germany is the country(name) and world is the table

  • this returns the population of Germany


    • You can use operators like this:

    SELECT ..., .../... FROM ... WHERE ...>...

    e.g. SELECT name, gdp/population FROM world WHERE area > 5000000

  • operators divide and greater than have been used

  • this returns the countries name and result of gdp/populationy


    • You can use AND to ensure that two or more conditions hold true:

    e.g. SELECT name, continent FROM world WHERE population < 2000000 AND gdp > 50000000000

  • this returns country names and continents that meet both conditions


    • You can use IN to check if an item is in a list:

    e.g. SELECT name, population FROM world WHERE name IN ('Ireland', Iceland', 'Demark')

  • this will return the listed contries name and population


    • The word LIKE permits pattern matching and % is the wildcard

    e.g. SELECT name FROM world WHERE name LIKE 'D%'

  • this retuns countries who's name begins with a D


    • The word BETWEEN allows for range checks:

    e.g. SELECT name, area FROM world WHERE area BETWEEN 250000 and 300000

  • this returns countries that have an 300000 > area > 250000


    • ORDER BY ...

    e.g. ORDER BY name

  • will return the query in order according to name


    • The UNDERSCORE '_' represents a single character wildcard

    e.g. '_t%'


    SELECT within SELECT:

    e.g. SELECT name FROM world WHERE population > (SELECT population FROM world WHERE name = 'Russia')

  • this returns all the countries whose population is bigger than Russia's


    • AGGREGATION FUNCTIONS

    SUM, Count, MAX, DISTINCT and GROUP BY


    SUM will give us the total of a given field

    e.g. SELECT SUM(population) FROM world


    the SELECT DISTINCT statement returns only distinct (different) values

    e.g. SELECT DISTINCT(continent) FROM world


    If you want to know how many of something meet your criteria you use COUNT

    e.g. SELECT Count(name) FROM world WHERE area > 1000000


    GROUP BY, HAVING is a little more tricky so here is a general plugin:

    SELECT column_name, aggregate_function(column_name)

    FROM table_name

    WHERE column_name operator value

    GROUP BY column_name

    HAVING aggregate_function(column_name) operator value;

    e.g. SELECT continent, COUNT(name) FROM world WHERE population > 10000000 GROUP BY continent HAVING COUNT(name) > 0;


    INNER JOIN

    JOIN is used to combine rows from two or more tables, based on a common field

    SELECT column_name

    FROM table_name

    INNER JOIN table2_name

    ON (id = common_id_name);

    e.g. SELECT team1,team2, player FROM game INNER JOIN goal ON (id=matchid) WHERE player LIKE 'Mario%'


    That's all for now! Looking forward to expanding on my SQL knowledge.
